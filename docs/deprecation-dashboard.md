# Deprecation Dashboard Report

This document defines output artifacts generated by `npm run deprecation:scan`.

## Outputs

- `reports/deprecation/deprecation-report.json`
- `reports/deprecation/deprecation-report.md`

Both files are uploaded by CI as the `deprecation-report` artifact.

## JSON schema

- `generatedAt`: ISO8601 datetime
- `summary`:
  - `filesScanned`
  - `ruleCount`
  - `findingCount`
- `byRule[]`:
  - `ruleId`
  - `description`
  - `severity`
  - `replacement`
  - `count`
- `findings[]`:
  - `ruleId`
  - `file`
  - `line`
  - `column`
  - `snippet`
  - `replacement`

## Dashboard usage

- Use `summary.findingCount` as top-level KPI.
- Group by `byRule` to track migration progress per deprecated API.
- Link `findings.file:line:column` from reports into PR review tasks.

## Operations

- To scan locally: `npm run deprecation:scan`
- To fail locally when findings exist: `npm run deprecation:scan:strict`
- Rule definitions live in `config/deprecations.json`.
