# Storybook Docs Template

Use this template for every new component and major pattern.

## 1. Overview
- Problem statement
- Intended users
- Supported use cases

## 2. API Surface
- Props and default values
- Controlled/uncontrolled behavior
- Event contract (`onChange`, `onSubmit`, etc.)

## 3. Do / Don't
- Minimum 3 Do rules
- Minimum 3 Don't rules
- Rules must be testable and specific

## 4. Accessibility
- Keyboard behavior
- Focus order and focus-visible states
- Screen reader labels and role semantics
- Color contrast assumptions

## 5. Interaction Test Set (Required)
- Happy path interaction
- Validation/error interaction
- Keyboard-only interaction
- Loading/empty/error state transition (if applicable)

## 6. Visual Regression Baseline
- Default state
- Critical variant state
- Error/warning state (if applicable)

## 7. Migration Notes
- Backward compatibility status
- Deprecated props (if any)
- Replacement path
